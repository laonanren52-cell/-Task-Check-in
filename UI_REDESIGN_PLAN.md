# SummerFlow UI / UX 深度重构计划

## 目标

将 SummerFlow 调整为一个高频、长期可用的个人学习工作台：任务流优先、信息可以快速扫读、复盘容易理解，且不牺牲原有的 Local First 数据与桌面端能力。

## 设计语言

- **气质**：雾白画布、深森林绿文字、冷灰蓝数据色、铜棕提醒色、低饱和灰紫专注色；轻边框与柔和表面层级替代大阴影和大渐变。
- **字体层级**：页面标题 42–56px；区块标题 24–30px；卡片标题 16–18px；正文 15–16px；表格与辅助说明 13–14px；标签 12–13px；数据指标 26–30px。
- **布局**：桌面端使用克制的宽工作区、细侧边栏和信息带；移动端改为单列自然文档流，底部导航固定。
- **动效**：页面淡入、浮层短位移、卡片 hover、面板切换；全部遵循 `prefers-reduced-motion`。

## 本轮落地

1. **今日页**：保持页面整体自然滚动。任务列表不再是内部滚动容器；任务卡支持长标题、长成果和多条目连续显示。
2. **任务交互**：更多菜单通过 Portal 挂载至 `document.body`，根据触发按钮实时定位，避免被卡片、表格或容器裁切。删除确认弹窗同样采用 Portal、Esc、焦点约束与触发点回焦。
3. **每日复盘**：重组为“今日状态 → 今日复盘 → 保存草稿 / 正式打卡”。1–5 分有明确情绪文字，不再出现没有解释的裸数字；复盘字段在桌面端是紧凑的 2 列工作区。
4. **数据看板**：收束为“节奏、去向、质量与连续性”三段叙事。将通用饼图换成横向投入条形图，并统一坐标轴、Tooltip、色彩与空状态。
5. **设置页**：保留已实现的二级导航结构；常用项目置前，数据与危险操作隔离在专用区块。

## 验收清单

- 首页在 2、5、10 条任务以及长文本条件下无重叠、裁切或菜单遮挡。
- 弹窗覆盖桌面侧栏、主内容和移动导航，Esc 可关闭，关闭后焦点回到原触发按钮。
- 宽屏复盘保持一行状态 + 两列字段；小屏改为单列，按钮可触达。
- 看板没有伪造数据；无记录时显示空状态。
- `npm run test`、`npm run build` 通过；不改变 IndexedDB 名称、数据结构或 Tauri identifier。
