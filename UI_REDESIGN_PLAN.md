# SummerFlow UI/UX 重构计划

## 目标

将 SummerFlow 从数据面板式界面调整为面向长期个人使用的学习工作台。重构优先级为：今日页、任务流、复盘、设置页，再统一所有页面的可读性和响应式行为。

## 设计语言

- 字体：Geist 与 Noto Sans SC；数据使用 Geist Mono 回退。
- 画布：雾白与纸感灰；主要文字采用深森林墨绿；信息辅助色使用雾霾蓝、铜棕、暗酒红、低饱和灰紫。
- 结构：低阴影、细分隔、嵌套表面，而非大面积卡片堆砌。
- 动效：页面渐入、任务入场、菜单展开、折叠展开、数字和进度过渡；遵从 reduced motion。

## 信息架构

1. 今日：概览、任务流、复盘、正式打卡处在同一文档流中。
2. 任务：筛选与批量操作置顶，桌面表头粘性，移动端使用卡片。
3. 设置：使用本页二级导航，分为软件更新、暑期目标、主题项目、学习科目、任务模板、数据备份、关于诊断。

## 关键修复

- 将基础文字提升至 15px；辅助、标签和表格文字不低于 13px。
- 任务卡取消固定式内部滚动，使用自适应网格和自然页面滚动；菜单提高到明确的浮层层级，长文本可换行。
- 复盘拆分为“今日状态”“今日复盘”“正式打卡”，1–5 分改为带文字解释的状态按钮。
- 设置区只呈现当前选中的模块，危险操作自然隔离在数据与备份中。

## 验收场景

- 今日页测试 2、5、10 项任务；长标题、长成果与菜单同时展开。
- 空状态、移动宽度、长表单字段、设置二级导航与折叠动画。
- `npm run test`、`npm run build` 以及本地视觉检查。
