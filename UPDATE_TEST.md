# 自动更新与数据保留测试说明

## 已自动化验证

- 旧版 HTML `summer-growth-v3` → 当前 IndexedDB。
- 0.1.0 → 0.1.1 设置迁移。
- 0.1.1 → 0.2.0 数据迁移版本升级。
- IndexedDB v2 → v3 后任务和设置保留。
- 非法语义版本和降级被拒绝。
- JSON 导入失败不覆盖现有数据。

运行：

```bash
npm run test
```

## 本地桌面验证矩阵

1. 安装 0.1.0，创建任务并正式打卡。
2. 完全退出再启动，确认任务、复盘和打卡存在。
3. 导出 JSON 到自选目录，再导入并核对计数。
4. 发布 0.1.1 测试 Release，确认 0.1.0 检测到更新。
5. 确认更新说明、发布时间和真实下载进度。
6. 下载完成后手动点击“立即重启并安装”。
7. 确认版本变为 0.1.1，原任务和打卡仍存在。
8. 确认更新前备份位于应用数据目录的 `backups` 子目录。

真实 GitHub Release 更新只有在 Secrets 配置并发布两个版本后才能完成。未完成前不得把此链路标记为“真实更新已验证”。
